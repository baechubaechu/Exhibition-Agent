# arduino-test — NeoPixel 스트립

## 스트립을 **중간에 잘라 멀리 이어 붙일 때** (LED 개수는 그대로)

WS2812 스트립은 **데이터가 LED를 차례로 지나가는** 구조입니다.  
잘라도 **총 LED 개수가 같으면** 펌웨어 `NUM_LEDS`(기본 60)는 **바꿀 필요 없습니다.**

### 잘라야 하는 위치

- LED **칸과 칸 사이** 절단선(보통 3패드: **5V, GND, DIN/DOUT**).
- 화살표 방향(**DIN → … → DOUT**)을 잃지 않기.

### 멀리 떨어진 두 덩어리를 **3가지 선**으로 이음

| 선 | 앞 조각 (ESP 쪽) | 뒤 조각 (멀리 있는 쪽) |
|----|------------------|------------------------|
| **데이터** | 잘린 면의 **DOUT** (또는 마지막 LED 출력) | **DIN** (화살표 들어가는 쪽) |
| **5V** | 5V 패드 | 5V 패드 |
| **GND** | GND 패드 | GND 패드 |

```
[ESP]──DIN──►[조각 A LED들]──DOUT ──긴 선──► DIN──[조각 B LED들]──►
        │                              │
        5V/GND (와고·굵은선) ────────────┴── 5V/GND (뒤 조각에도 꼭!)
```

- **DOUT → DIN** 만 이으면 됨. 5V끼리, GND끼리 각각 이어야 함.
- **듀폰은 데이터(DOUT→DIN) 짧은 구간**에만. **5V/GND는 굵은 선 + 와고** (거리 길면 더 중요).

### 전원 (멀리 떨어뜨릴 때 필수)

- **앞·뒤 조각 모두**에 5V/GND 공급(**전원 보강**) — 한쪽만 먹이면 뒤 조각이 안 켜지거나 흔들림.
- **GND는 ESP·어댑터·앞·뒤 조각 전부 한 덩어리.**
- 거리 **1m 이상**이면 5V/GND **굵은 선(0.5mm² 이상)**, 가능하면 **양 조각 입구**에 각각 와고.

### 거리·신호

- 데이터 선 **너무 김**(수 m) → 색 튐·꺼짐 → **짧게**, **DIN 앞 330Ω**, 필요 시 **레벨 시프터(3.3V→5V)**.
- 실험: `test.ino` → `1` (앞 구간) → `99` (전체) → 뒤 조각까지 밝는지 확인.

### 안 되면 점검

1. **DOUT↔DIN** 반대로 안 꽂았는지 (화살표 방향).
2. **뒤 조각 5V/GND** 빠졌는지.
3. 이음 납땜·와고 **흔들림** (이전에 꺼지던 것과 동일).
4. **총 LED 수**가 여전히 60개인지 — 중간에 LED를 **물리적으로 빼 버린** 게 아니라 **선만 늘린** 경우 코드 수정 없음.

## 스케치 선택

| 폴더 | 용도 |
|------|------|
| `test/` | 시리얼 1~12, **연속 60LED** 12구간 (A·일반 스트립) |
| `test_matrix_b/` | **Matrix B** — **96LED / 12모형**, 시리얼 1~12 = 단면, 99=전체, 100=지도 |
| `esp32_http_light_matrix/` | **Matrix B 전시** — Wi-Fi HTTP, GPIO13, 노트북 `EXHIBITION_LIGHT_MATRIX_HTTP_URL` |
| `esp32_http_light_plan_a/` | **전체모형 전시** — 64LED Wi-Fi HTTP, `EXHIBITION_LIGHT_HTTP_URL` |
| `esp32_http_light/` | 구형 60LED 일자 스트립 (WPA2 개인 Wi-Fi) |
| `esp32_http_light_ent/` | WPA2-Enterprise |

Matrix B LED 번호·모형 매핑: `matrix_b_layout.h` (시리얼 `100` 으로 출력)

데이터 핀: Matrix B **`GPIO13`**, 전체모형·일반 스트립 **`GPIO12`** ↔ DIN.
