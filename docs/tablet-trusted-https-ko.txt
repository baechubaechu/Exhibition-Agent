================================================================================
  태블릿에서 HTTPS “안전하지 않음” 경고 줄이기 (LAN 전시)
================================================================================
  Next 기본 dev HTTPS(--experimental-https)는 자체 서명이라 Chrome 이
  새로고침·재접속 시마다 경고를 띄울 수 있습니다. 서버 옵션만으로 끌 수 없고,
  **태블릿이 신뢰하는 CA 로 서명된 인증서**를 쓰면 해결됩니다.

  가장 간단한 방법: PC 에 mkcert 설치 → 같은 LAN IPv4 가 들어간 인증서 발급
  → 태블릿에 mkcert 루트 CA 설치 → 레포 루트의 start-dev-trusted.bat 사용.

--------------------------------------------------------------------------------
  1) Windows PC — mkcert
--------------------------------------------------------------------------------
  · Chocolatey 예:  choco install mkcert
  · 또는 https://github.com/FiloSottile/mkcert/releases

  관리자 아님 일반 PowerShell 에서:

       mkcert -install

  노트북 IPv4를 ipconfig 로 확인한 뒤 (예: 192.168.0.7):

       cd certs
       mkcert 127.0.0.1 localhost 192.168.0.7

  생성된 두 파일을 아래 이름으로 복사·이름 맞춤 (레포 루트의 `certs\` 경로):

       certs\cert.pem       ← mkcert 가 만든 *.pem (공개)
       certs\cert-key.pem   ← *-key.pem (비밀)

  *.pem 은 Git 에 올리지 마세요.

--------------------------------------------------------------------------------
  2) 안드로이드 태블릿 — mkcert 루트 CA 설치
--------------------------------------------------------------------------------
  PC 에서 mkcert 가 저장한 루트 위치 확인:

       mkcert -CAROOT

  그 폴더의 rootCA.pem 을 태블릿으로 옮긴 뒤:

       설정 → 보안 → 암호화 및 자격 증명 → 인증서 설치 → CA 인증서
       (기기·제조사마다 메뉴 문구는 조금 다름)

  한 번 설치해 두면 같은 mkcert 로 만든 사이트는 **연결 경고 없이** 열리는
  경우가 많습니다. (기기 정책·브라우저에 따라 다를 수 있음)

--------------------------------------------------------------------------------
  3) 실행
--------------------------------------------------------------------------------
  레포 루트에서 **start-dev-trusted.bat** 실행.
  (일반 start-dev.bat 대신 — mkcert 인증서로 Next HTTPS 기동)

  FastAPI 는 이때 **EVENT_BRIDGE_SSL_VERIFY 를 끄지 않음**(PC 에도 mkcert -install 한 경우
  Windows 가 해당 인증서를 신뢰하는 전제).

  만약 uvicorn 이 브리지 HTTPS 에 연결하지 못하면: PC 에 mkcert -install 했는지 확인하거나,
  임시로 exhibition-agent 실행 전에만 EVENT_BRIDGE_SSL_VERIFY=false 로 시험.

--------------------------------------------------------------------------------
  4) 여전히 경고가 나오면
--------------------------------------------------------------------------------
  · cert.pem 에 **태블릿에서 치는 것과 동일한 호스트**가 들어갔는지 확인
    (IP 로 접속하면 인증서 SAN 에 그 IP 가 있어야 함)
  · 태블릿 브라우저가 **설치한 사용자 CA** 를 신뢰하는지(학교 정책 등)
  · 임시로만 쓸 거면 키오스크·전용 브라우저 정책은 별도 검토
